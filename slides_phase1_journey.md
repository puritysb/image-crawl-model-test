# 웹 이미지 크롤러 & 모델 테스트 프레임워크: Phase 1 개발 여정

---

## 1. Project Overview & Goals

### 제목: 웹 이미지 크롤러 & 모델 테스트 프레임워크: 프로젝트 개요 및 목표

### 핵심 내용:
*   프로젝트의 궁극적인 목표: 키워드 기반 이미지 수집 및 AI 모델 테스트 자동화 프레임워크 구축.
*   해결하려는 핵심 문제: AI 모델 검증을 위한 대용량 테스트 데이터 수집 및 다양한 모델 API 테스트의 비효율성.
*   주요 목표: 유연한 이미지 수집 엔진, 플러그인 기반 모델 테스트 시스템, 확장 가능한 아키텍처, 통합 분석 대시보드.
*   기대 효과: 개발 생산성 향상, 일관된 벤치마킹, 빠른 프로토타이핑, 연구 가속화.

### 스피커 노트:
*   "이 프로젝트는 AI 모델 검증 과정에서 겪는 데이터 수집 및 테스트의 비효율성을 해결하기 위해 시작되었습니다."
*   "우리의 목표는 다양한 이미지 소스에서 데이터를 자동으로 수집하고, 여러 AI 모델을 체계적으로 테스트할 수 있는 범용적인 프레임워크를 구축하는 것입니다."
*   "이를 통해 개발 생산성을 높이고, 모델 성능을 객관적으로 비교하며, 새로운 아이디어를 빠르게 검증할 수 있을 것으로 기대합니다."

## 2. Technical Architecture Journey

### 제목: Phase 1 기술 아키텍처 여정

### 핵심 내용:
*   **하이브리드 아키텍처**: Python 백엔드 (이미지 수집, 모델 테스트)와 Next.js 프론트엔드 (웹 대시보드) 구성.
*   **모듈형 및 플러그인 기반 디자인**: 유지보수성 및 확장성 확보.
*   **주요 기술 스택**: Python, JavaScript/TypeScript, FastAPI, Next.js, Supabase.
*   **이미지 수집 전략**: Pixabay, Pexels, Unsplash, Google Custom Search API 우선 사용, API 제한 시 Scrapy를 활용한 직접 크롤링 백업.
*   **데이터 저장**: Supabase (PostgreSQL)를 중앙 데이터 저장소로 활용.
*   **디자인 패턴**: Strategy, Factory, Observer 패턴 적용.

### 스피커 노트:
*   "Phase 1에서는 Python과 Next.js를 활용한 하이브리드 아키텍처를 구축하여 시스템의 기반을 다졌습니다."
*   "특히, 다양한 이미지 소스를 유연하게 연동하기 위해 API 기반 수집을 우선하고, 필요시 웹 크롤링을 백업으로 사용하는 전략을 채택했습니다."
*   "Supabase를 통해 이미지 메타데이터와 크롤링 작업 상태를 효율적으로 관리하며, 모듈형 및 플러그인 기반 디자인으로 향후 확장성을 고려했습니다."

## 3. Key Challenges & Solutions

### 제목: Phase 1 주요 도전 과제 및 해결 방안

### 핵심 내용:
*   **도전 과제 1: 다양한 이미지 검색 API 연동 및 관리**:
    *   해결 방안: 각 API별 클라이언트 모듈화 (`crawler/api_sources`), 환경 변수를 통한 API 키 관리.
*   **도전 과제 2: 대용량 이미지 메타데이터 저장 및 관리**:
    *   해결 방안: Supabase (PostgreSQL)를 활용한 안정적인 데이터 저장, 효율적인 스키마 설계 (`image_metadata`, `crawl_jobs`).
*   **도전 과제 3: 백엔드와 프론트엔드 간의 비동기 작업 통신**:
    *   해결 방안: FastAPI 백엔드에서 `subprocess`를 통해 크롤링 작업을 별도 프로세스로 실행, Supabase 실시간 구독을 통한 프론트엔드 상태 업데이트.
*   **도전 과제 4: 크롤링 방식 변경에 따른 기존 코드 재설계**:
    *   해결 방안: 기존 Scrapy 관련 파일 제거 및 API 기반 수집 로직으로 전면 재설계, 필요한 부분만 재활용.

### 스피커 노트:
*   "Phase 1을 진행하면서 여러 기술적 난관에 부딪혔습니다. 특히 다양한 이미지 API를 통합하고 대용량 데이터를 효율적으로 관리하는 것이 중요했습니다."
*   "저희는 API 클라이언트를 모듈화하고 Supabase를 중앙 데이터 저장소로 활용하여 이 문제를 해결했습니다."
*   "또한, 백엔드와 프론트엔드 간의 비동기 통신을 위해 `subprocess`와 Supabase 실시간 구독을 활용하여 사용자 경험을 개선했습니다."
*   "초기 크롤링 방식 변경이라는 큰 결정이 있었지만, 이를 통해 더욱 유연하고 확장 가능한 시스템을 구축할 수 있었습니다."

## 4. Development Milestones

### 제목: Phase 1 개발 마일스톤: 핵심 기능 구현

### 핵심 내용:
*   **프로젝트 초기화 및 아키텍처 설정**: 하이브리드 아키텍처 기반 디렉토리 구조, Supabase DB 설정, 환경 변수 구성.
*   **Next.js 웹 대시보드 기본 구성**: 기본 레이아웃, 네비게이션, Supabase 연동, 메인 페이지 및 크롤러 페이지 UI 구현.
*   **Python 백엔드 (FastAPI) 개발**: FastAPI 애플리케이션 설정, `/api/crawl` 및 `/api/crawl/jobs` 엔드포인트 구현, `subprocess`를 통한 크롤링 작업 실행.
*   **이미지 검색 API 클라이언트 구현**: Pixabay, Pexels, Unsplash, Google Custom Search API 클라이언트 개발.
*   **이미지 수집 기능 구현 및 테스트**: 대시보드에서 키워드 기반 이미지 수집 시작, Supabase에 데이터 저장 확인.
*   **이미지 갤러리 페이지 구현**: 수집된 이미지 그리드 표시, 필터링, 정렬, 이미지 다운로드 기능.

### 스피커 노트:
*   "Phase 1에서는 프로젝트의 핵심 기반을 다지는 데 집중했습니다."
*   "Next.js 대시보드와 FastAPI 백엔드를 구축하고, Supabase를 통해 데이터 흐름을 통합했습니다."
*   "특히, 다양한 이미지 검색 API를 연동하여 키워드 기반 이미지 수집 기능을 성공적으로 구현했으며, 수집된 이미지를 편리하게 관리하고 다운로드할 수 있는 갤러리 페이지까지 완성했습니다."
*   "이러한 마일스톤 달성을 통해 프로젝트의 핵심 가치를 실현할 수 있는 기반을 마련했습니다."

## 5. Deployment Success

### 제목: Phase 1 배포 성공: 안정적인 서비스 운영 기반 마련

### 핵심 내용:
*   **배포 플랫폼**: Next.js 애플리케이션은 Vercel에 배포.
*   **데이터베이스**: Supabase를 클라우드 기반 데이터베이스로 활용하여 안정적인 데이터 관리.
*   **환경 변수 관리**: `.env` 파일을 통해 API 키 및 민감 정보 안전하게 관리.
*   **성공적인 기능 확인**:
    *   대시보드 기본 UI 및 Supabase 연동 확인.
    *   크롤러 페이지에서 키워드 기반 이미지 수집 작업 시작 및 상태 업데이트 확인.
    *   이미지 갤러리 페이지에서 수집된 이미지 확인 및 다운로드 기능 검증.

### 스피커 노트:
*   "Phase 1에서 개발된 시스템은 Vercel과 Supabase를 활용하여 성공적으로 배포되었습니다."
*   "이를 통해 안정적인 서비스 운영 환경을 확보했으며, 개발된 모든 핵심 기능들이 실제 환경에서 정상적으로 동작함을 확인했습니다."
*   "특히, 키워드 기반 이미지 수집부터 갤러리에서 이미지 확인 및 다운로드까지의 전체 워크플로우가 원활하게 작동하는 것을 검증했습니다."

## 6. Phase 2 Roadmap

### 제목: 다음 단계: Phase 2 로드맵

### 핵심 내용:
*   **플러그인 시스템 구현**: 다양한 AI 모델 API를 쉽게 연동할 수 있는 플러그인 아키텍처 개발.
*   **QR Detector 예시 구현**: 첫 번째 모델 테스트 플러그인으로 QR Detector 통합.
*   **테스트 시나리오 관리 시스템**: 모델 테스트를 위한 시나리오 정의 및 관리 기능 개발.
*   **분석 도구 및 대시보드 확장**: 테스트 결과 분석 및 시각화 기능 강화 (Phase 3와 연계).
*   **문서화**: 개발자 가이드 및 API 문서 작성 시작.

### 스피커 노트:
*   "Phase 1에서 이미지 수집 엔진의 기반을 다졌다면, Phase 2에서는 이 프로젝트의 핵심 가치인 '모델 테스트 프레임워크'를 본격적으로 구현할 예정입니다."
*   "플러그인 시스템을 통해 다양한 AI 모델을 유연하게 연동하고, QR Detector를 첫 번째 예시로 통합하여 실제 활용 사례를 보여줄 것입니다."
*   "또한, 테스트 시나리오 관리 및 결과 분석 기능을 고도화하여 사용자들이 모델 성능을 더욱 심층적으로 이해할 수 있도록 지원할 계획입니다."
*   "이러한 다음 단계를 통해 프로젝트의 완성도를 높이고, AI 모델 검증의 새로운 표준을 제시할 것입니다."
